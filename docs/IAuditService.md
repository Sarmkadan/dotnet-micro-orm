# IAuditService

Provides an asynchronous interface for recording and querying audit information within the dotnet‑micro‑orm framework. Implementations persist insert, update, delete and failure operations, expose counters for monitoring, and allow retrieval of logs by various criteria.

## API

### AuditService  
**Type:** `AuditService` (property)  
**Purpose:** Returns the concrete audit service instance associated with the interface, useful when the implementation needs to expose additional members not defined in `IAuditService`.  
**Return Value:** An object of type `AuditService`.  
**Exceptions:** None.

### LogInsertAsync  
**Signature:** `public Task<AuditLog> LogInsertAsync(object entity, CancellationToken cancellationToken = default)`  
**Purpose:** Records an insert operation for the supplied entity.  
**Parameters:**  
- `entity` – The instance being inserted; must not be `null`.  
- `cancellationToken` – Optional token to cancel the operation.  
**Return Value:** An `AuditLog` representing the persisted audit entry.  
**Exceptions:**  
- `ArgumentNullException` if `entity` is `null`.  
- `OperationCanceledException` if the token is triggered.  
- Any exception thrown by the underlying storage provider.

### LogUpdateAsync  
**Signature:** `public Task<AuditLog> LogUpdateAsync(object originalEntity, object updatedEntity, CancellationToken cancellationToken = default)`  
**Purpose:** Records an update operation, capturing the state before and after the change.  
**Parameters:**  
- `originalEntity` – The entity as it existed before the update; must not be `null`.  
- `updatedEntity` – The entity after the update; must not be `null`.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** An `AuditLog` for the update.  
**Exceptions:**  
- `ArgumentNullException` if either entity argument is `null`.  
- `OperationCanceledException` if cancellation is requested.  
- Storage‑related exceptions as appropriate.

### LogDeleteAsync  
**Signature:** `public Task<AuditLog> LogDeleteAsync(object entity, CancellationToken cancellationToken = default)`  
**Purpose:** Records a delete operation for the supplied entity.  
**Parameters:**  
- `entity` – The instance being deleted; must not be `null`.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** An `AuditLog` for the deletion.  
**Exceptions:**  
- `ArgumentNullException` if `entity` is `null`.  
- `OperationCanceledException` if cancellation occurs.  
- Provider‑specific exceptions.

### LogFailureAsync  
**Signature:** `public Task<AuditLog> LogFailureAsync(Exception exception, object entity = null, CancellationToken cancellationToken = default)`  
**Purpose:** Logs a failed operation, associating the exception and optionally the entity involved.  
**Parameters:**  
- `exception` – The exception that caused the failure; must not be `null`.  
- `entity` – Optional entity related to the failure.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** An `AuditLog` capturing the failure details.  
**Exceptions:**  
- `ArgumentNullException` if `exception` is `null`.  
- `OperationCanceledException` if the token is canceled.  
- Any exception from the audit store.

### GetAuditLogsAsync  
**Signature:** `public Task<List<AuditLog>> GetAuditLogsAsync(int? skip = null, int? take = null, CancellationToken cancellationToken = default)`  
**Purpose:** Retrieves audit log entries, optionally paginated.  
**Parameters:**  
- `skip` – Number of entries to skip (for paging).  
- `take` – Maximum number of entries to return.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** A list of `AuditLog` objects matching the criteria.  
**Exceptions:**  
- `ArgumentOutOfRangeException` if `skip` or `take` is negative.  
- `OperationCanceledException` on cancellation.  
- Storage exceptions.

### GetUserActivityAsync  
**Signature:** `public Task<List<AuditLog>> GetUserActivityAsync(string userId, int? skip = null, int? take = null, CancellationToken cancellationToken = default)`  
**Purpose:** Returns audit logs generated by a specific user.  
**Parameters:**  
- `userId` – Identifier of the user; must not be `null` or empty.  
- `skip` – Optional skip count for paging.  
- `take` – Optional take count for paging.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** List of `AuditLog` entries for the user.  
**Exceptions:**  
- `ArgumentException` if `userId` is invalid.  
- `ArgumentOutOfRangeException` for negative paging values.  
- `OperationCanceledException` if canceled.  
- Provider exceptions.

### GetLogsByActionAsync  
**Signature:** `public Task<List<AuditLog>> GetLogsByActionAsync(string action, int? skip = null, int? take = null, CancellationToken cancellationToken = default)`  
**Purpose:** Retrieves logs filtered by the action type (e.g., “Insert”, “Update”, “Delete”).  
**Parameters:**  
- `action` – The action string to filter on; must not be `null` or empty.  
- `skip` – Optional skip count.  
- `take` – Optional take count.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** List of matching `AuditLog` entries.  
**Exceptions:**  
- `ArgumentException` if `action` is invalid.  
- `ArgumentOutOfRangeException` for negative paging values.  
- `OperationCanceledException` on cancellation.  
- Storage exceptions.

### GetFailedOperationsAsync  
**Signature:** `public Task<List<AuditLog>> GetFailedOperationsAsync(int? skip = null, int? take = null, CancellationToken cancellationToken = default)`  
**Purpose:** Returns audit logs that represent failed operations.  
**Parameters:**  
- `skip` – Optional skip count.  
- `take` – Optional take count.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** List of `AuditLog` entries where the operation failed.  
**Exceptions:**  
- `ArgumentOutOfRangeException` for negative paging values.  
- `OperationCanceledException` if canceled.  
- Provider exceptions.

### GetLogsByDateRangeAsync  
**Signature:** `public Task<List<AuditLog>> GetLogsByDateRangeAsync(DateTimeOffset start, DateTimeOffset end, int? skip = null, int? take = null, CancellationToken cancellationToken = default)`  
**Purpose:** Retrieves logs whose timestamps fall within the supplied inclusive range.  
**Parameters:**  
- `start` – Inclusive start of the range.  
- `end` – Inclusive end of the range; must be greater than or equal to `start`.  
- `skip` – Optional skip count.  
- `take` – Optional take count.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** List of `AuditLog` entries within the date range.  
**Exceptions:**  
- `ArgumentException` if `start` is after `end`.  
- `ArgumentOutOfRangeException` for negative paging values.  
- `OperationCanceledException` on cancellation.  
- Storage exceptions.

### PurgeOldLogsAsync  
**Signature:** `public Task<int> PurgeOldLogsAsync(TimeSpan olderThan, CancellationToken cancellationToken = default)`  
**Purpose:** Deletes audit logs older than the specified timespan relative to the current UTC time.  
**Parameters:**  
- `olderThan` – Timespan; logs with a timestamp earlier than `UtcNow - olderThan` are removed. Must be positive.  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** The number of log entries removed.  
**Exceptions:**  
- `ArgumentOutOfRangeException` if `olderThan` is zero or negative.  
- `OperationCanceledException` if cancellation occurs.  
- Provider exceptions.

### GetSummaryAsync  
**Signature:** `public Task<AuditSummary> GetSummaryAsync(CancellationToken cancellationToken = default)`  
**Purpose:** Returns a summary object containing aggregated counts of operations.  
**Parameters:**  
- `cancellationToken` – Optional cancellation token.  
**Return Value:** An `AuditSummary` instance with fields such as total, successful, failed, inserts, updates, deletes.  
**Exceptions:**  
- `OperationCanceledException` if canceled.  
- Provider exceptions.

### DisposeAsync  
**Signature:** `public async ValueTask DisposeAsync()`  
**Purpose:** Asynchronously releases resources held by the audit service (e.g., closes database connections).  
**Return Value:** A `ValueTask` that completes when disposal is finished.  
**Exceptions:** None expected; any exception indicates a failure to clean up resources.

### TotalOperations  
**Type:** `int` (read‑only property)  
**Purpose:** Cumulative count of all audit‑logged operations (inserts, updates, deletes, failures) since the service started or was last reset.  
**Exceptions:** None.

### SuccessfulOperations  
**Type:** `int` (read‑only property)  
**Purpose:** Number of operations that completed successfully (inserts, updates, deletes).  
**Exceptions:** None.

### FailedOperations  
**Type:** `int` (read‑only property)  
**Purpose:** Number of operations that resulted in a failure and were logged via `LogFailureAsync`.  
**Exceptions:** None.

### Inserts  
**Type:** `int` (read‑only property)  
**Purpose:** Count of insert operations logged.  
**Exceptions:** None.

### Updates  
**Type:** `int` (read‑only property)  
**Purpose:** Count of update operations logged.  
**Exceptions:** None.

### Deletes  
**Type:** `int` (read‑only property)  
**Purpose:** Count of delete operations logged.  
**Exceptions:** None.

## Usage

### Example 1: Logging and retrieving activity
```csharp
await using var audit = serviceProvider.GetRequiredService<IAuditService>();

// Log an insert
var newEntity = new Customer { Name = "Contoso" };
await audit.LogInsertAsync(newEntity);

// Log an update
var existing = await db.Customers.FindAsync(newEntity.Id);
existing.Name = "Contoso Ltd";
await audit.LogUpdateAsync(existing, existing);

// Retrieve all logs for the current user
var userLogs = await audit.GetUserActivityAsync(User.Identity.Name);
foreach (var log in userLogs)
{
    Console.WriteLine($"{log.Timestamp}: {log.Action} on {log.TableName}");
}
```

### Example 2: Obtaining a summary and cleaning up old records
```csharp
await using var audit = serviceProvider.GetRequiredService<IAuditService>();

// Get a summary of operations since startup
var summary = await audit.GetSummaryAsync();
Console.WriteLine($"Total: {summary.TotalOperations}, " +
                  $"Successful: {summary.SuccessfulOperations}, " +
                  $"Failed: {summary.FailedOperations}");

// Remove logs older than 90 days
int removed = await audit.PurgeOldLogsAsync(TimeSpan.FromDays(90));
Console.WriteLine($"Purged {removed} stale audit entries.");
```

## Notes

- All logging methods throw `ArgumentNullException` when a required reference argument is `null`. Implementations should validate arguments before performing any I/O.
- The counter properties (`TotalOperations`, `SuccessfulOperations`, etc.) are updated atomically by the implementation; however, callers should not rely on them being a instantaneous snapshot if multiple threads are logging concurrently—values may reflect a short‑lived intermediate state.
- `DisposeAsync` must be called (or the instance used with `await using`) to ensure that underlying connections or file handles are released. After disposal, further calls to any member of the service may throw `ObjectDisposedException`.
- Pagination arguments (`skip`, `take`) are interpreted as zero‑based offsets; negative values result in `ArgumentOutOfRangeException`. If both are `null`, the implementation may return all matching records, which could be large—callers should consider using paging for extensive data sets.
- Date‑range queries treat the `start` and `end` values as inclusive; timestamps stored with higher precision (e.g., ticks) are compared using `>= start` and `<= end`.
- The service is thread‑safe for concurrent logging operations; implementations should synchronize internal state or use concurrent collections to guarantee correct counter updates and log ordering. Read‑only query methods do not modify state and can be safely invoked concurrently with writes.
