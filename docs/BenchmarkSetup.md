# BenchmarkSetup

BenchmarkSetup provides helper members for configuring the test environment and representing a benchmark entity in the **dotnet-micro-orm** project. The static members manage the DI container and database lifecycle, while the instance members map to the columns of a benchmark test entity.

## API

### GetServiceProvider
- **Purpose:** Retrieves the `IServiceProvider` that holds the services configured for the benchmark harness (e.g., DbContext, logging).
- **Parameters:** None.
- **Return value:** An `IServiceProvider` instance.
- **Exceptions:** Throws `InvalidOperationException` if the service provider has not been initialized by a prior call to `InitializeDatabaseAsync`.

### InitializeDatabaseAsync
- **Purpose:** Asynchronously creates the database schema and seeds any required baseline data for the benchmarks.
- **Parameters:** None.
- **Return value:** A `Task` that completes when the database is ready.
- **Exceptions:** 
  - `SqlException` (or provider‑specific equivalent) if the connection cannot be opened or schema creation fails.
  - `InvalidOperationException` if the method is called while a database is already initialized.

### CleanupDatabaseAsync
- **Purpose:** Asynchronously drops the test database tables and removes all data, returning the environment to a clean state.
- **Parameters:** None.
- **Return value:** A `Task` that completes when cleanup is finished.
- **Exceptions:** 
  - `SqlException` if the connection fails during teardown.
  - `InvalidOperationException` if called before the database has been initialized.

### CreateTestEntityAsync
- **Purpose:** Asynchronously creates a new `BenchmarkTestEntity` instance with default or random values, persists it to the database, and returns the populated entity.
- **Parameters:** None.
- **Return value:** A `Task<BenchmarkTestEntity>` representing the newly created entity.
- **Exceptions:** 
  - `DbUpdateException` if the insert operation fails.
  - `InvalidOperationException` if the database has not been initialized.

### Id
- **Purpose:** Stores the primary key of the benchmark entity.
- **Parameters:** None.
- **Return value:** `int`.
- **Exceptions:** None (value is typically generated by the database on insert).

### Name
- **Purpose:** Holds a short, readable identifier for the entity.
- **Parameters:** None.
- **Return value:** `string`.
- **Exceptions:** The setter may throw `ArgumentNullException` if a `null` value is assigned.

### Value
- **Purpose:** Contains an integer metric associated with the benchmark.
- **Parameters:** None.
- **Return value:** `int`.
- **Exceptions:** None.

### Description
- **Purpose:** Provides an optional, longer explanation of the entity.
- **Parameters:** None.
- **Return value:** `string` (may be `null` or empty).
- **Exceptions:** The setter may throw `ArgumentException` if the string exceeds the column length limit defined in the model.

### CreatedDate
- **Purpose:** Records the UTC timestamp when the entity was first persisted.
- **Parameters:** None.
- **Return value:** `DateTime`.
- **Exceptions:** None.

### ModifiedDate
- **Purpose:** Records the UTC timestamp of the last update to the entity; may be `null` if the entity has never been modified.
- **Parameters:** None.
- **Return value:** `DateTime?`.
- **Exceptions:** None.

## Usage

```csharp
// Obtain the service provider and prepare the database
var provider = BenchmarkSetup.GetServiceProvider;
await BenchmarkSetup.InitializeDatabaseAsync();

// Create a test entity and inspect its properties
var entity = await BenchmarkSetup.CreateTestEntityAsync();
Console.WriteLine($"Id: {entity.Id}, Name: {entity.Name}, Value: {entity.Value}");

// Tear down the database after the benchmark run
await BenchmarkSetup.CleanupDatabaseAsync();
```

```csharp
// Manually instantiate and populate a BenchmarkSetup entity = new BenchmarkSetup
{
    Name = "Sample",
    Value = 123,
    Description = "A sample benchmark entity",
    CreatedDate = DateTime.UtcNow,
    ModifiedDate = DateTime.UtcNow
};

// After saving changes via DbContext, Id will be populated by the database
```

## Notes
- The static members (`GetServiceProvider`, `InitializeDatabaseAsync`, `CleanupDatabaseAsync`, `CreateTestEntityAsync`) share global state; calling them concurrently from multiple threads may lead to race conditions. It is recommended to serialize access or ensure that initialization/cleanup occurs on a single thread before parallel benchmark execution.
- `ModifiedDate` is nullable to reflect entities that have not been updated after insertion; consumers should check for `null` before using the value.
- Setters for `Name` and `Description` enforce basic validation (non‑null for `Name`, length limits for `Description`) and will throw the corresponding exceptions if invalid data is supplied.
- Instance properties do not perform any automatic synchronization with the underlying database; values must be persisted explicitly via a `DbContext` or similar ORM mechanism.
